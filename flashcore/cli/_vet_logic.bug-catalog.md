# Bug Catalog for _vet_logic.py

## Overview
The module `flashcore/cli/_vet_logic.py` contains validation logic for flashcard decks. A known bug is that `_validate_and_normalize_card` incorrectly retains the `s` (score) field when mapping card data, causing a `ValidationError` because the `Card` model forbids the `s` attribute.

## Bugs

### Bug 1: Retention of `s` field during validation
- **Description**: `_validate_and_normalize_card` maps `q` to `front` and `a` to `back` but does not remove the `s` (score) field from the dict before constructing a `Card`.
- **Blast radius**: Any YAML card file that includes a valid `s` field (score) will cause the vetting command to fail with a false validation error, preventing the card from being written back to disk.
- **Why plausible**: The parser in `parser.py` correctly `pop`s the `s` field, but `_vet_logic.py` mirrors the mapping logic without the pop, leading to inconsistent behavior.
- **Test type**: Captured bug / contract pin (unit test that feeds a card dict containing `s` and expects a `ValidationError` to be *absent*; the test should fail because the bug is present).
